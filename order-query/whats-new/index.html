What's New

Updates to Katana Order Query — flexible sales order and returns reporting for Katana MRP.

v1.2.0 30 Mar 2026 Latest
  • Tab-memory API key — key is never written to disk or storage; it lives only in the open tool tab and is gone the moment the tab is closed or refreshed
  • Activation overlay — on opening the tool, an activation screen prompts for your API key; it validates against Katana before unlocking the tool
  • Return Status filter — filter returns by status (Not Returned / Returned / Returned All) directly in the Returns scope
  • SKU count on multi-line orders — shows 3 SKUs instead of listing all SKUs in a single cell
  • Qty formatting fixed — quantities no longer display with excessive decimal places
  • Return status labels humanized — RETURNED_ALL now displays as Returned, NOT_RETURNED as Not Returned
  • Footer bar no longer obscures the Run Query button in the sidebar
v1.1.0 27 Mar 2026 Previous
  • Added Returns scope — query sales return records with expandable line item detail (SKU, qty returned, return reason, batch #, restock location, refund status)
  • Added API Panel and Debug Panel in the footer bar — toggle on to inspect every API call made and query lifecycle events
  • Version badge in header links to this page
  • Context-aware popup: new users without an API key see a Get Started button linking to setup documentation
  • Documentation link in footer bar
v1.0.0 15 Mar 2026 First Release
  • Query open, closed, or all sales orders with flexible field selection
  • Filters for customer, order number, date ranges, status, currency, ship-from location, and item/SKU
  • Collapsible line item rows for multi-item orders
  • Sortable columns and resizable column widths
  • Export to CSV
  • Totals bar for QTY and value summaries
