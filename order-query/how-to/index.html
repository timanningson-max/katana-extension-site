How To Use

Katana Order Query lets you flexibly search, filter, and export your Katana sales orders and returns — including fields not available in the standard Katana interface.

1 · Setup

1
Install the extension
Load the extension folder in Chrome via chrome://extensions → Load unpacked. Enable Developer mode first if it is not already on.
2
Add your Katana API key
Click the extension icon, then click ⚙ Settings. Paste your Katana API key and save.
Where to find your API key In Katana: Settings → Integrations → API → Generate API key.
3
Open the tool
Click the extension icon, then click 📋 Sales Order Query. The tool opens in a new browser tab.

2 · Running a Query

1
Choose a scope
Use the scope buttons at the top of the sidebar to select what to query: Open (active orders), All (everything), Closed (delivered), or Returns (sales returns).
2
Select your display fields
Tick the fields you want as columns. Fields are grouped into Order, Row (line items), and Fulfillment sections. The 10 default fields are pre-selected — add or remove as needed.
Tip Selecting Row or Fulfillment fields triggers additional API calls per order to resolve that data. On large result sets this can take a moment.
3
Apply filters
Narrow results using the filters below the field selector: customer name, order number, date ranges, status, currency, ship-from location, and more.
4
Click Run Query
Results appear in the table. Orders with multiple line items show a summary row — click the ▶ arrow to expand individual line items.

3 · Querying Returns

Switch to the Returns scope to query sales return records. Returns have their own set of display fields — Return No, Customer, Status, Original SO, line item details, refund status, and more.

  • Filter by Return Date range and Return Status (Not Returned / Returned / Returned All)
  • Tick row-level fields (SKU, Item, Qty Returned, Return Reason, Batch #) to see line item detail — returns with multiple items show an expandable row
  • The Original SO column shows the source sales order number, not the internal ID

4 · Exporting to CSV

Click Export CSV in the top-right after running a query. The export contains one row per line item (expanded), with all currently selected display fields as columns. Date fields are exported in ISO format for easy sorting in Excel or Google Sheets.

5 · Tips

  • API Panel and Debug Panel are available in the footer bar — toggle them on to see exactly which API calls were made and how many records were returned at each step. Useful for troubleshooting slow queries.
  • Column sorting — click any column header to sort. Click again to reverse. Date columns sort chronologically, not alphabetically.
  • Column resizing — drag the right edge of any column header to resize it.
  • Search across results — use the "Search across results" box at the bottom of the sidebar to filter the already-loaded results by any visible field value without re-running the query.
  • Results per page — increase this to 100 or 200 for large exports. The CSV always exports the full result set regardless of page size.
  • Rate limiting — Katana's API allows 60 requests per 60 seconds. The tool automatically retries if it hits the limit, but very large queries with many resolution fields may be slow. The API Panel shows if any 429 (rate limited) responses occurred.
