What's New

Release history for the Katana Order Generator Chrome extension — an interim replacement for Katana AI (KAI) while native functionality is rebuilt.

v1.2.1 7 March 2026 Latest

Bug Fixes

  • Qty / price now extracted from the correct table row — the split-SKU scanner previously used a flat context window around the matched SKU position, causing two systematic errors: (1) a 30-character pre-window bled the previous row's unit price in, shifting all prices down by one row; (2) pack-count numbers embedded in product descriptions (e.g. (box/100 pcs)) were matched as the order quantity before the actual QTY column value. The scanner now looks up the matching Y-grouped visual row and extracts qty/price from there — the same clean, isolated row that the line-by-line parser uses.
v1.2.0 6 March 2026

Improvements

  • PDF table extraction rebuilt — pdf.js text items are now grouped by Y coordinate into proper visual rows before parsing (previously all items were joined into one flat blob). Each table row becomes its own line, so quantity and price are always extracted from the correct row — not contaminated by numbers from other rows or product descriptions.
  • Split-SKU scan preserved — the whitespace-tolerant hyphenated SKU scan continues to run on the original flat content stream, so SKU fragments that are adjacent in column-major PDFs (e.g. ALU-3MM- 1020) are still found.
  • False positive removed — removed a loose SKU-match fallback that produced incorrect matches (e.g. BC-Bulk matching unrelated text). Tight matching only.
v1.1.0 6 March 2026

Improvements

  • PDF line item matching — SKUs that wrap across lines in PDF tables (e.g. ALU-3MM-1020 split as ALU-3MM- 1020) are now found correctly using a whitespace-tolerant scan. Unit prices and quantities are extracted from the surrounding context automatically.
  • Customer matching — now checks the customer's company name and individual first/last name fields in addition to the Katana display name, so company-only records are matched correctly.
  • SKU false positives reduced — SKU matching is now case-sensitive and requires a minimum of 3 characters, preventing short or common-word SKUs from matching prose text in emails.
  • Order item false positives reduced — word-based (low confidence) matching now requires at least 3 significant words to match, preventing short product names from triggering on natural language sentences.
  • PO reference formats — added support for Reference Number: 1234 and reference number; 1234 patterns.
Known issue Products in the Katana catalog with very short or common-word names (e.g. a product literally named "price" or "Table") may still produce false positive matches. This is a catalog quality issue — removing or renaming those products resolves it.
v1.0.0 5 March 2026 Initial Release
  • Paste text or upload a PDF to extract customer, PO reference, delivery date, and line items
  • Three-tier product matching: by SKU (high confidence), product name (medium), keywords (low)
  • Editable review screen before submitting to Katana
